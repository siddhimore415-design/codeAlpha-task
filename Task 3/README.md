Pizaa-saless-Dashboard
🚀 Excited to share my final internship project — Pizza Sales Dashboard using Power BI 🍕📊

This project focuses on Exploratory Data Analysis (EDA) and interactive business intelligence reporting to uncover valuable insights from pizza sales data.

🔍 Exploratory Data Analysis (EDA)
During this project, I performed detailed EDA by:

✔ Asking meaningful business questions before analysis ✔ Exploring dataset structure, variables, and data types ✔ Identifying sales trends, customer ordering patterns, and anomalies ✔ Testing assumptions using statistics and visualizations ✔ Detecting possible data quality issues for further analysis

📌 Key KPIs Created using DAX
💰 Total Revenue
Calculated total sales revenue generated from all pizza orders.

Total Revenue = SUM(pizza_sales[total_price])
🧾 Average Order Value
Calculated average customer spending per order.

Avg Order Value = [Total Revenue] / [Total Order]
🍕 Total Pizzas Sold
Calculated total quantity of pizzas sold.

Total Pizza Sold = SUM(pizza_sales[quantity])
📦 Total Orders
Calculated total number of unique customer orders.

Total Order = DISTINCTCOUNT(pizza_sales[order_id])
📈 Average Pizzas Per Order
Measured customer purchasing behavior.

Avg Pizza per Order = [Total Pizza Sold] / [Total Order]

📌 Key Insights Derived
📍 Classic category contributes maximum sales and orders 📍 Large-size pizzas generate highest revenue 📍 Weekend orders are significantly higher than weekdays 📍 July and January recorded peak monthly sales 📍 Thai Chicken Pizza emerged as top-selling pizza 📍 Brie Carre Pizza showed lowest sales performance 📍 Customers order an average of 2–3 pizzas per transaction
